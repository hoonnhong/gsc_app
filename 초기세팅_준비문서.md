# 업무 관리 앱 초기 세팅 준비 가이드

성공적인 앱 개발과 실행을 위해, 프로젝트 폴더(`e:\dev\gsc_app`)를 중심으로 다음 단계의 세팅을 진행하시기 바랍니다.

## 1. 파이썬 가상환경 및 필수 패키지 설치
앱은 Python 환경에서 동작하므로 모듈 간 충돌을 방지하기 위해 가상환경(`venv`) 구성이 필수입니다. 터미널(PowerShell 등)을 열고 아래 명령어를 실행하세요.

```powershell
# 프로젝트 경로로 이동
cd e:\dev\gsc_app

# 가상환경 생성 및 활성화
python -m venv venv
.\venv\Scripts\activate

# 필수 패키지 설치
pip install nicegui pandas gspread google-auth-oauthlib google-api-python-client openpyxl python-dotenv
```
*(추후 BigQuery 연동 시 `google-cloud-bigquery` 여부 결정 필요)*

## 2. 구글 클라우드 플랫폼(GCP) 인증 세팅
데이터베이스로 구글 시트를 사용하기 때문에, 앱이 시트에 접근할 수 있도록 권한을 부여하는 과정입니다.

1. **[GCP 콘솔](https://console.cloud.google.com/)**에 접속하여 새 프로젝트(예: `GSC-App-Project`)를 생성합니다.
2. 라이브러리 메뉴에서 **Google Sheets API**와 **Google Drive API**를 각각 검색하여 **활성화(Enable)** 합니다.
3. **서비스 계정(Service Account)** 탭으로 이동하여 새 서비스 계정을 생성합니다.
4. 생성된 계정의 '키(Keys)' 메뉴에서 **새 키 만들기(JSON 형태)**를 선택하여 다운로드 받습니다.
5. 다운로드 받은 JSON 파일을 프로젝트 내부 경로(예: `e:\dev\gsc_app\secrets\gcp_service_account.json`)로 이동시킵니다.
   > ⚠️ **주의:** 이 JSON 파일은 절대 Github이나 외부 클라우드에 노출되면 안 됩니다. `e:\dev\gsc_app\.gitignore` 파일에 경로를 반드시 추가하세요.

## 3. 베이스 구글 시트 템플릿 준비
로직에서 바탕이 될 시트들입니다. 시트를 새로 만든 후, **위 2번에서 만든 서비스 계정 이메일(예: `app-bot@gsc-app.iam.gserviceaccount.com`)을 열어 '편집자' 권한으로 공유를 완료해주세요.**

### A. 시스템 설정 시트 (동적 메뉴용)
- **시트명 권장:** `gsc_시스템_설정`
- **시트 탭 1열(A~I) 구성:** `menu_id`, `parent_id`, `menu_level`, `menu_name`, `icon`, `route`, `allowed_roles`, `sort_order`, `is_active`

### B. 개발 이슈 트래커 시트
- **시트명 권장:** `gsc_앱개발_이슈관리`
- **시트 탭 1열(A~I) 구성:** `issue_id`, `유형`, `상태`, `우선순위`, `제목`, `상세내용`, `요청자`, `등록일`, `완료일`

*(회계나 기타 업무용 로직 시트는 개발 진행 과정에서 순차적으로 추가합니다.)*

## 4. 환경 변수 (.env) 설정
사용자 환경에 맞게 키를 맵핑하기 위해, 프로젝트 루트(`e:\dev\gsc_app`)에 `.env` 파일을 생성하고 아래 내용을 각자의 시트 상황에 맞게 붙여넣습니다.

```env
# 구글 인증키 경로 지정
GOOGLE_APPLICATION_CREDENTIALS=secrets/gcp_service_account.json

# 시스템 설정 시트 ID (메뉴 동적 생성용)
SYSTEM_SHEET_ID=여기에_시스템_설정_시트_ID를_넣으세요

# 이슈 트래커 시트 ID
ISSUE_TRACKER_SHEET_ID=여기에_이슈트래커_시트_ID를_넣으세요
```

## 5. 권장되는 앱 폴더 구조
작업을 효율적으로 진행하기 위해 다음과 같은 골격의 디렉토리 배치를 목표로 개발을 진행하게 됩니다. 직접 미리 폴더를 생성해주시면 좋습니다.

```text
gsc_app/
│
├── .env                  # 환경 변수 목록 (시트 ID 등)
├── .gitignore            # Git 관리에 제외할 파일 명시 (secrets/ 등)
├── main.py               # 단일 포털 앱 진입점 메인 파일
├── config.py             # .env 로더 및 전역 공통 시스템 환경 변수
│
├── core/                 # 핵심 아키텍처 로직 폴더
│   ├── gcp_connector.py  # 구글 시트 연동 통합 기능
│   └── ui_layout.py      # 동적 좌측 사이드바/상단 탭 구조 설정
│
├── modules/              # 쪼개진 단위 앱 모듈 파일
│   ├── accounting.py     # 회계 관련 코드
│   └── hr.py             # 직원/인사 관련 코드
│
└── secrets/              # GCP 인증서 파일 위치 (보안 폴더)
    └── gcp_service_account.json
```

위 5단계를 마치셨다면 앱의 뼈대(Skeleton) 코드를 작성할 모든 기본적인 준비가 완료된 것입니다.
